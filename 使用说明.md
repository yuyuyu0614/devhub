# DevHub 使用说明

## 🚀 快速开始

### 安装和启动

1. **安装依赖** (首次运行):
   ```bash
   cd devhub
   npm install
   ```

2. **启动应用**:
   ```bash
   npm start
   ```
   
   或者双击 `start.bat` (Windows)

### 首次设置

1. 应用启动后，会显示设置窗口
2. 填写以下信息:
   - **API Base URL**: `https://api.example.com/v1`
   - **API Key**: 您的 Cloud API Key
   - **默认模型**: 如 `your-model-name` 或 `your-model-flash`
   - **Token 预算**: 设置日/月 Token 使用上限

## 🎯 核心功能

### 1. 项目管理
- **添加项目**: 点击顶部 "+" 按钮，输入项目名称和本地路径
- **切换项目**: 使用顶部下拉框选择不同项目
- **项目设置**: 每个项目可设置独立的默认模型

### 2. 会话管理
- **新建会话**: 在中间面板点击 "新建会话"
- **重命名会话**: 右键点击会话名称
- **删除会话**: 右键点击会话名称
- **切换会话**: 点击会话名称

### 3. AI 聊天
- **发送消息**: 在底部输入框输入消息，按 `Ctrl+Enter` 发送
- **模型选择**: 在顶部状态栏选择不同的模型（Cloud 云端 / Ollama 本地）
- **消息历史**: 所有聊天记录自动保存
- **本地模型**: 支持 Ollama 本地模型（如 qwen2.5、qwen3、llama3.1），自动检测已安装模型并加入下拉列表

### 4. 任务监控
- **实时输出**: 切换到 "📊 任务监视器" 标签页
- **任务解析**: 自动识别文件操作、命令执行等任务类型
- **历史记录**: 查看所有任务历史，支持搜索和过滤

### 5. Token 用量
- **实时统计**: 顶部状态栏显示当前会话、今日、本月 Token 用量
- **详细报表**: 切换到 "📋 Token 用量" 标签页查看图表和表格
- **预算提醒**: 达到预算阈值时弹窗提醒

### 6. Skills 管理
- **查看 Skills**: 点击左侧栏 "Skills 管理"
- **添加 Skill**: 在 .claude/skills/ 目录下创建 .md 文件
- **编辑 Skill**: 点击编辑按钮，使用系统默认编辑器打开

### 7. Agents 管理
- **查看 Agents**: 点击左侧栏 "Agents 管理"
- **添加 Agent**: 在 .claude/agents/ 目录下创建 .md 文件
- **编辑 Agent**: 点击编辑按钮，使用系统默认编辑器打开

## 🎨 界面操作

### 侧边栏
- **折叠/展开**: 点击左上角的菜单按钮
- **项目列表**: 显示所有已添加的项目
- **管理入口**: Skills 和 Agents 管理入口

### 顶部状态栏
- **项目选择**: 下拉框选择当前项目
- **模型选择**: 下拉框选择当前使用的模型
- **Token 统计**: 显示会话、今日、本月 Token 用量
- **主题切换**: 右侧按钮切换深色/浅色主题

### 标签页
- **💬 聊天**: 主聊天界面
- **📊 任务监视器**: 实时任务监控
- **📋 Token 用量**: Token 统计和图表

## ⚙️ 配置说明

### 全局设置
存储在 `~/.devhub-data/settings.json`:
```json
{
  "apiBaseUrl": "https://api.example.com/v1",
  "apiKey": "your-api-key",
  "defaultModel": "your-model-name",
  "dailyTokenBudget": 1000000,
  "monthlyTokenBudget": 30000000,
  "theme": "light"
}
```

### 项目配置
存储在 `~/.devhub-data/projects.json`:
```json
[
  {
    "name": "项目名称",
    "path": "/path/to/project",
    "defaultModel": "your-model-flash"
  }
]
```

### 会话数据
存储在 `~/.devhub-data/sessions/<项目名>/<会话ID>.json`

### Token 用量数据
存储在 `~/.devhub-data/usage.json`

## 🔧 常见问题

### 1. 应用无法启动
- 检查 Node.js 是否安装: `node --version`
- 检查依赖是否安装: 确保 `node_modules` 目录存在
- 查看控制台错误信息

### 2. API 调用失败
- 确保 Cloud API Key 正确填写
- 检查网络连接是否正常（需访问 `api.Cloud.com`）
- 确认 API Key 有足够额度
- 查看控制台错误信息定位具体原因

### 3. Ollama 本地模型连接失败
- 确保 Ollama 已启动（端口 `11434`）
- 检查模型是否已安装：`ollama list`
- 未安装时可运行：`ollama pull qwen2.5:1.5b` 等
- 计算机内存需大于模型大小才能正常运行

### 4. Token 用量不显示
- 检查 `~/.devhub-data/usage.json` 文件是否存在
- 确认文件权限正确
- 查看控制台是否有错误信息

### 5. 项目无法添加
- 检查项目路径是否存在
- 确认路径有读取权限
- 项目名称不能重复

### 6. 主题切换无效
- 刷新页面或重启应用
- 检查 CSS 文件是否正确加载

## 📁 文件结构

```
devhub/
├── main.js              # Electron 主进程
├── preload.js           # Context Bridge 预加载脚本
├── package.json         # 项目配置和依赖
├── renderer/            # 渲染进程文件
│   ├── index.html      # 主界面 HTML
│   ├── style.css       # 样式文件
│   ├── app.js          # 主应用逻辑
│   └── task-parser.js  # 任务解析模块
├── start.bat           # Windows 启动脚本
├── README.md           # 项目说明
└── 使用说明.md         # 本文件
```

## 🔒 安全注意事项

1. **API Key 安全**: API Key 存储在本地 JSON 文件中，请确保文件安全
2. **数据备份**: 定期备份 `~/.devhub-data/` 目录
3. **文件权限**: 确保应用有足够的文件读写权限
4. **网络连接**: 应用需要网络连接调用 Cloud API

## 📞 技术支持
